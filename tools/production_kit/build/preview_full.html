SECTION A · SPECIFICATION

신규 규칙 제작서

사명/비전 합성 골격(RESPONSE-DIRECT), 5-단 엔진 파이프라인, 슬롯 매핑, 정규화 가드 — v4.1 production rule의 단일 진실 공급원(SSoT).

A-1. 사명/비전 골격 (현행 · RESPONSE-DIRECT)

과거 7-slot/3-Tier 템플릿 골격은 폐기되었습니다. 현재는 synthMissionVisionFromResponses()가 고객 응답(Q13 가치·Q63 판단기준·Q75 분야·Q39/Q41 활동·Q73 성취)을 100% 직접 합성해 헤드라인 한 문장을 만듭니다(유형 템플릿 제거). 근거 안내문구는 데이터에만 보존(화면 미노출), 다이어리 본문은 빈 문자열로 미사용입니다.

계층수식 (formula)상태/예시
헤드라인 (화면 노출)<strong>Mission</strong>: 고객 응답(Q13 가치 · Q63 판단기준 · Q75 분야 · Q39/Q41 활동 · Q73 성취)에서 직접 합성한 &quot;사람들이 …하도록 돕는&quot; 핵심 구절.<br><strong>Vision</strong>: 같은 응답에서 합성한 &quot;누구나 …사는 세상, 그 한가운데서 …살아간다&quot; 형태의 회상 정체성 구절.🎯 문제를 분석해 해법을 찾는 힘으로 사람들이 자기다운 삶을 선택하도록 돕는다<br>🌅 누구나 자기답게 사는 세상, 그 한가운데서 자기 길을 열어 가며 살아간다
근거 안내문구 (미노출)<strong>Subline/Footer</strong>: 🔍 활동 응답([Q39/Q41 actLabel])과 가치·관심 분야 응답을 기반으로 도출되었습니다.<span class="mono" style="font-size:8.5pt">[PR-중복제거] 데이터 보존 · 화면 렌더 제거</span>

A-2. 정규화 가드 (한국어 자연성)

한국어 합성에서 자주 발생하는 어색함을 빌더 단계에서 자동 차단합니다. 모든 가드는 정규식 + 받침 검사(_hangulJong)로 결정적으로 동작합니다.

가드패턴규칙적용 위치
의문형 명사절 조사 차단<code>/(는지|을지|할지|런지|을까|는가|할까)$/</code>명사절 어미가 의문형으로 끝나면 "을/를" 첨가하지 않음<span class="mono" style="font-size:8.2pt">_whyNatural() — report-engine-v4.js</span>
`특히 특히` 중복 차단<code>/(특히\s+){2,}/g → &quot;특히 &quot;</code>topic_scene 라이브러리는 이미 "특히 ~" 시작이므로 외부에서 "특히" 추가 금지<span class="mono" style="font-size:8.2pt">_scenePrefix() + 메인 사명 빌더 — report-engine-v4.js</span>
비전 헤드라인 인용 정리<code>/\s*(?:으로|로)\s*기억된다\s*$/</code>인용("...") 안에서 명사구만 노출되도록 종결부 제거<span class="mono" style="font-size:8.2pt">_stripVisionHeadlineTail() — program-engine.js</span>
도메인 결합 와/과 조사<code>_hangulJong(primary) &gt; 0 ? &quot;과 &quot; : &quot;와 &quot;</code>받침 있으면 "과", 없으면 "와" — 예술과 미디어 / 경제와 교육<span class="mono" style="font-size:8.2pt">domainPhrase 합성부 — program-engine.js</span>
으로/로 받침 검사<code>_hangulJong(word) === 0 || jong === 8 ? &quot;로&quot; : &quot;으로&quot;</code>받침 없거나 ㄹ받침이면 "로", 그 외 "으로"<span class="mono" style="font-size:8.2pt">_josa_eulo() — report-engine-v4.js</span>
마침표 인용 정리<code>/[.。!?！？]+$/</code>인용 부호 안 들어가는 헤드라인은 끝마침표 자동 제거<span class="mono" style="font-size:8.2pt">_stripTrailingPunct() — program-engine.js</span>
근거 안내문구 화면 미노출 [PR-중복제거]<code>missionSubline / visionSubline / footer 렌더 호출 제거</code>근거 안내문구("🔍 활동 응답…")는 데이터에는 보존하되 화면(report.html)에는 렌더하지 않음 — 사명/비전 카드 3회 반복 차단<span class="mono" style="font-size:8.2pt">mission_vision 렌더 case — report.html</span>
헤드라인 재진술 차단 [PR-중복제거]<code>core-line(coreOneLine) / mv-aux(한 줄 통합본) 렌더 호출 제거</code>type-line과 거의 같은 core-line, 헤드라인과 동일한 "당신의 사명/비전:" 통합본은 화면에서 제거 — 한 문장으로 충분<span class="mono" style="font-size:8.2pt">summary 헤더 + mission_vision 렌더 case — report.html</span>
다음 단계 제안 = 사이클 완주 이후 가이드 [PR-#5]<code>nextSteps = [{이 사이클을 마치면}, {다음 사이클은 이렇게}]</code>본문 타임라인(3주/3개월/1년)을 반복하지 않고, 1년 사이클 완주 이후 다음 사이클 가이드 2문장만 합성(primaryDomain·약축결·visionHeadline)<span class="mono" style="font-size:8.2pt">§6 nextSteps 빌드 — program-engine.js</span>

A-3. 5-단 엔진 + 합성 파이프라인 (9 Step)

리포트·프로그램은 아래 5개 엔진을 순차로 거쳐 생성됩니다. 본 매뉴얼의 모든 기술은 이 5-단 엔진의 현행 동작과 cross-check되었습니다.

엔진진입 함수역할
① report-engine.jsRE.build()4축 점수·강점·진로/교육 기본 골격
② report-engine-v4.jsV4.upgrade()톤 해상도·도메인 확장·RESPONSE-DIRECT 사명/비전 합성
③ career-engine.jscareerRules진로/경력/교육 큐레이션·추천 확장 방향
④ program-engine.jsPE.build()분기·모듈·실행·다음 단계 제안(사이클 완주 이후 가이드)
⑤ report.html / program.html렌더중복제거 정책(근거문구·재진술 미노출) 적용한 화면 출력
  1. Step 1. 56문항 응답 → fingerprint 산출 (P2-1)
  2. Step 2. Q6 trait 정규화 → 4축 점수 계산 → tier 라벨(deep/active/sharp/quiet) 부여
  3. Step 3. Q13 가치 → primaryCategory(관계/자유/성장/원칙지향) 결정 → 톤 후보 생성
  4. Step 4. 톤 우선순위 해상도(P1-1) — 4축 강세 + Q63 compass + Q41 topic 가중
  5. Step 5. Q75 도메인 → primary × secondary 확장(p×s = 21×21=441 path) (P1-2)
  6. Step 6. Q63 Compass(9 카테고리) → 합성 키워드 결정
  7. Step 7. [현행] synthMissionVisionFromResponses() — 사명/비전 헤드라인을 고객 응답 100%로 직접 합성(유형 템플릿 폐기). 근거 안내문구는 데이터에만 보존(화면 미노출), 다이어리 본문은 미사용.
  8. Step 8. 진로/교육 다양성 가드(P1-3) + 추천 확장 방향(Q63·Q75·tone seed 고유 합성) + 17 QA 통과 → 최종 리포트
  9. Step 9. [프로그램] PE.build() — 분기/모듈/실행 + 다음 단계 제안(1년 사이클 완주 이후 가이드 2문장, primaryDomain·약축결·visionHeadline 합성)

                      SECTION D · MANUAL TEMPLATE PROMPT

                      수동 제작용 템플릿 프롬프트

                      사람도 AI도 동일 결과를 만들 수 있도록 정리한 단일 프롬프트 + 슬롯 입력 양식 + few-shot. LLM에게는 그대로 system message로, 사람에게는 작성 매뉴얼로 사용합니다.

                      D-1. System Prompt (LLM 전용)

                      아래 텍스트를 LLM의 system 메시지에 그대로 붙여 넣으세요. 가드 위반 시 출력을 거부하도록 설계되어 있습니다.

                      당신은 인생포트폴리오 v4.1 리포트 합성기입니다. 아래 슬롯과 가드 규칙을 엄격히 따라, 진단 응답에서 곧바로 사명/비전 헤드라인을 합성(RESPONSE-DIRECT)합니다. 고정 분류 라벨이나 미리 정해 둔 골격 문장을 끼워 넣지 않습니다.

[입력 슬롯]
- Q13 values         : 핵심 가치 1~3개 (예: 사랑, 자유, 의미 추구)
- Q63 compass        : 선택 기준 1~2개 (선택의 why)
- Q75 domains        : 관심 분야 1~3개 (primary × secondary)
- Q41 topic_scene    : "특히 ~"로 시작하는 장면 한 줄
- 4축 점수           : self_understanding/self_expression/self_design/self_execution
- 톤(top priority)   : principled_designer / warm_connector / visionary_creator / pragmatic_achiever / reflective_explorer
- name               : 사용자 이름

[출력 골격 — RESPONSE-DIRECT]
· missionHeadline : 응답에서 합성한 "지금 무엇을 하는 사람인가" 한 문장. "…한다."로 끝남.
    형태: "[강점·방식]으로 [Q75 대상]이 [Q13×Q63이 바라는 변화]를 하도록 돕는다/만든다" 류
    예: "문제를 분석해 해법을 찾는 힘으로 사람들이 자기다운 삶을 선택하도록 돕는다"
· visionHeadline  : 응답에서 합성한 "앞으로 무엇이 되려 하는가" 한 문장. "…된다/…한다."로 끝남.
· missionSubline / visionSubline : 화면 미노출(근거 안내문구). 출력하더라도 리포트에는 렌더되지 않음.

[합성 가드 — 위반 시 다시 출력]
1. 의문형 명사절(하는지/하는가/하는지를/하는가를)에는 "을/를" 절대 금지. 예: "왜 이 일을 하는지" (O), "왜 이 일을 하는지를" (X)
2. 자모 분리 결합 금지. "더하ㄴ다"(X) → "더한다"(O). "는 것" 종결은 반드시 완성형 음절로 합성.
3. 헤드라인은 인용("...") 안에 들어가도 자연스러운 한 문장. 군더더기·재진술 금지.
4. 도메인 결합 와/과: 받침 있으면 "과", 없으면 "와". 예: "예술과 미디어" / "경제와 교육".
5. 으로/로: 받침 없거나 ㄹ받침이면 "로", 그 외 "으로". 예: "의미로", "신뢰로", "원칙으로".
6. 을/를: 받침 있으면 "을", 없으면 "를". 자동 보정.
7. 마크다운(**) 사용 금지. 일반 텍스트만.
8. 평이체(0-A) 원칙: 추상 문학어 대신 일상 행동 동사로. "나침반 삼아"·"곁에 있어주는" 같은 상투어 금지, 명사형 정지어는 동사형 진행어로.

[고유성]
- 동일 입력은 항상 동일 결과로 산출합니다(결정적). 80억 명이 진행해도 각자의 응답에서만 합성됩니다.
- 같은 라이프브릿지의 보정 선택은 fingerprint(56문항 응답 해시) 시드 기반 pickByHash()로 결정.

[출력 형식]
JSON으로만 응답하세요:
{
  "missionHeadline": "...",   // "…한다."
  "visionHeadline": "...",    // "…된다/…한다."
  "missionSubline": "...",    // 화면 미노출(근거)
  "visionSubline": "..."      // 화면 미노출(근거)
}

## 0. 핵심 발견 — 문서 전제 vs 실제 (⚠️ 배경 §0 수정 필요)

원문서 §0 배경은 **"(A)안으로 §3 응답직합성 전환 완료"** 를 전제로 하나, 실제 코드는 다르다.

| 원문서 전제 | 실제 코드 상태 | 판정 |
|---|---|---|
| "(A)안으로 §3 전환 완료·배포" | 실제는 **[P18b] B안**이 구현·배포됨 | ⚠️ **명칭 상이** — 결과(§3 직합성)는 이미 달성 |
| §3(주차 subline/if-then/actions/effects, 3개월, 1년) tone pool 렌더 중 | **이미 synthesize\* 로 우회 완료** (`weekSubline`, `synthesizeIfThenBlock`, `synthesizeWeekActions`, `synthesizeWeekEffects`, `synthesizeMonth3`, `synthesizeYear1`) | ✅ **완료됨** — 재작업 불필요 |
| tone 5종 pool = dead code 보존 | 사실. dead code 주석 + 롤백 보존 블록 존재 | ✅ 일치 |
| §3 밖(modules/effects/risks/quarter/closing) tone pool 잔존 | **부분 사실**: `tonePack.modules`(L1966) · `tonePack.effects`(L2040) · `tonePack.risks`(L2198) 아직 직접 참조 | ✅ **핵심 유효 과제** |
| 응답변수 매핑(강점/분야/나침반/활동/도구/환경/약축) | 코드에서 실제 사용 **102회** (compassKw 27·primaryDomain 23·userTool1 14·userActivities 11·userTopStrength 10·userWeakGrain 9·userFocusEnv 8) | ✅ 유효 |
| data-cue-type / data-outcome-\* 메타 | 코드·HTML에 **전무** | ❌ 신규 |
| 살아냄 규칙 5종(Gollwitzer/Stacking/Celebration/Examen/Rule of Life) 명시적 규칙화 | if-then은 **감정·상태 트리거만** 사용(시간/장소/사건 큐 구분 없음). 나머지 4종 미규칙화 | ❌ 신규 |

**결론:** 원문서 "축 A" 중 **§3 자체는 이미 완료**. 남은 것은 **§3 밖 3개 섹션(modules·effects·risks)의 tone pool 잔존 제거**뿐. "축 B"(살아냄 규칙 5종)는 대체로 신규지만, **현재 서베이·리포트에 이미 있는 변수/구조 위에서만** 얹을 수 있는 것으로 한정해야 함.

## 1. ✅ 지금 서베이·리포트에 "필요하고 가능한" 항목 (채택 권장)

아래는 **현재 응답 데이터·구조로 곧바로 구현 가능**하고, 서비스 품질을 실제로 높이는 것만 추린 목록이다.

### N1. §3 밖 잔여 tone pool 제거 (원문서 축 A의 실제 잔여분) — **우선순위 최상**
- **대상:** `modules`(L1966 `tonePack.modules`), `effects`(L2040 `tonePack.effects`), `risks`(L2198 `tonePack.risks`) 3개 섹션만.
  - (quarter/heading/board/closing은 추가 조사로 tone pool 직접 참조 여부 확정 후 포함 결정 — 현재 grep상 tonePack 직접 참조는 위 3개.)
- **이유:** DoD(1)("어떤 렌더 경로에서도 tone pool 노출 없음")을 문자 그대로 충족하는 **유일한 실질 잔여 과제**. 이미 §3에서 검증된 synthesize\* 패턴을 그대로 확장하면 됨.
- **가능성:** 응답변수 이미 확보(102회 사용) → 즉시 가능.

### N2. if-then ② 상황 큐 다양화 (Gollwitzer) — **부분 채택**
- **현행:** `_WHEN_STATE_KO`가 **감정·상태 큐**만 사용(예: "막막할 때", "지칠 때").
- **필요분:** 여기에 **시간·장소·사건 큐**를 변주 풀로 추가하면 실행 트리거가 구체화되어 살아냄 확률↑ (연구 근거 타당).
- **한정:** 원문서의 `data-cue-type` 메타 태깅은 **UI 미노출 전제이므로 실익이 낮음** → 우선 **문장 다양화만** 채택, 메타 태깅은 보류(§2 참조).
- **가능성:** `_whenTrigger()` 확장으로 가능. 활동/도구 택일 로직 유지.

### N3. Habit Stacking 문장 1개 (주차당) — **채택**
- **현행:** `synthesizeWeekActions`가 동사형 행동 3개 생성.
- **필요분:** 3개 중 1개를 "{기존 일상} 뒤에 {행동}" 구조로. 앵커는 **응답 우선(userActivities/userTool1/userFocusEnv)**, 없으면 일반 생활 앵커(fabricated 아님).
- **가능성:** 응답변수 이미 있음 → 가능. 대원칙-B(응답 결측 시 폴백) 준수.

### N4. 1년 문장 "Rule of Life" 3문장(사명·비전·강점 축) — **조건부 채택**
- **현행:** `synthesizeYear1` 존재. 문장 개수·축 구성은 재확인 필요.
- **필요분:** 정확히 3문장(사명 1·비전 1·강점 1), 각 응답변수 ≥3개 결합.
- **한정:** 현재 year1 출력 구조(program.html 렌더 호환)를 깨지 않는 선에서. **구조 변경이 크면 별도 스프린트로 분리.**

### N5. 주간 회고 4행 재정렬(Examen 뼈대) — **조건부 채택**
- **현행:** retro 4행 존재(L1044~1047). UI 라벨 유지 전제.
- **필요분:** 뼈대만 재정렬(Awareness/Intention/Review/Forward). **UI·라벨 무변경**이면 저위험.
- **한정:** 종교 언어 UI 미노출(§7 헌법과 동일). 개발자 코멘트에만 근원 표기.

## 2. ⏸️ 지금은 불필요/보류 항목 (현행 서베이·리포트 기준)

| 원문서 항목 | 보류 사유 |
|---|---|
| **data-cue-type / data-outcome-form / data-outcome-size 메타 태깅** | UI 미노출 전제 → 지금 당장 **사용처(전자다이어리·자산카드)가 없음**. 해당 기능 착수 시 함께. 지금 넣으면 미사용 속성만 늘어남(YAGNI). |

| **시각 언어 잠금 / 전자다이어리 / 자산카드** | 원문서도 이번 라운드 제외 명시. 동의. |
| **함수 대거 신설(synthesizeQuarterLead 등 7종)** | quarter/board/closing이 tone pool을 **직접** 참조하지 않으면 신설 불필요. **실측 후 최소 신설**(현재 확정 잔여는 modules/effects/risks 3개뿐). |

## 3. 🔒 현행 헌법과의 정합 (이미 지켜지는 것 — 재확인)

원문서 금지사항 대부분은 **이미 우리 헌법·코드가 준수** 중이다:
- tone pool 프로덕션 재활성화 금지 → ✅ 이미 dead code 우회
- 종교 언어 UI 미노출 → ✅ §7 원칙과 동일
- 게이미피케이션(streak/포인트/랭킹) 금지 → ✅ 브랜드 가드레일 일치
- 도메인 라벨 직접 노출 금지 → ✅ §7 + 대원칙-C
- fabricated data 금지, 결측 시 폴백 → ✅ 대원칙-B
- 리포트 엔진 대수술 금지(렌더 경로 우회만) → ✅ 기존 방침
- 결정론(같은 응답=같은 결과, Math.random 금지) → ✅ fingerprint/variantIdx

→ **추가로 성문화할 신규 규칙은 없음.** 대원칙-C(융합)·대원칙-D(카피)로 이미 커버.

## 4. 권장 착수 범위 (최소·정직 버전)

원문서 7커밋(P1.6-1~7)을 현실에 맞게 **압축**:

1. **[R1] §3 밖 tone pool 제거** — modules·effects·risks 3개만 synthesize\* 전환 (DoD(1) 완결). ★ 최우선·독립 커밋
2. **[R2] if-then 큐 다양화** — 시간/장소/사건 큐 변주 풀 추가(문장만, 메타 태깅 제외)
3. **[R3] Habit Stacking** — 주차 1문장 앵커 구조
4. **[R4] 1년 Rule of Life 3문장** — year1 구조 재확인 후, 안전하면 진행/크면 분리
5. **[R5] 주간 회고 Examen 재정렬** — UI 무변경 시에만

**제외(별도 스프린트):** data-\* 메타 태깅, Celebration form/size, 시각언어/전자다이어리/자산카드, quarter/board/closing 신설(잔존 미확정).

**검증(공통):** node --check → 엔진 테스트(§7=0·재현성 100%) → 분리 커밋(파일 단위) → push → Firebase 배포 → SHA256 local↔live → Playwright 콘솔 0 → 12섹션 앵커/인증 가드 정상.

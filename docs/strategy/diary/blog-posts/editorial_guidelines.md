# 블로그 편집 가이드라인 (Editorial Guidelines)

> **목적**: 인생포트폴리오 다이어리 콘텐츠 마케팅 12편의 톤·구조·진정성 기준을 통일.
> **핵심 원칙**: 독자가 주어. 출처 명시. 보편 현상으로 추상화.
> **마지막 수정**: 2026-05-19

---

## 0. 이 문서를 쓰는 이유

블로그는 "이걸 사세요"가 아니라 **"당신의 삶에 이 질문이 필요합니다"**라고 말해야 한다.
독자가 검색창에 적은 갈증을 정확히 받아내되, 광고처럼 보이지 않아야 한다.
정직함이 곧 차별화다. 모든 주장에는 **검증 가능한 출처**가 따라붙어야 한다.

---

## 1. 절대 원칙: "3 No, 3 Yes"

### ❌ 3 No (절대 등장하지 않는 것)

| 항목 | 이유 | 예시 |
|------|------|------|
| **① 저자 이야기** | 1인칭 회고는 자기계발 인플루언서의 영역. 본 블로그는 "독자 중심" 매체로 포지셔닝. | "제가 30대에…" / "저는 이렇게 극복했고…" |
| **② 브랜드명·서비스명** | "인생포트폴리오", "다이어리 상품명", 자체 검사 명칭을 본문에 직접 노출하지 않음. (CTA 영역 제외) | "저희 인생포트폴리오에서는…" / "다이어리를 사세요" |
| **③ 인플루언서·경쟁사명** | 특정 인물 의존은 신뢰 위험. 경쟁사 언급은 노이즈. | "유튜버 ○○○이 말한…" / "△△ 다이어리와 달리…" |

### ✅ 3 Yes (반드시 지키는 것)

| 항목 | 이유 | 적용법 |
|------|------|--------|
| **① 독자가 주어** | "당신", "우리", "사람들"로 시작. 독자가 자기 이야기로 읽게 함. | "당신이 새벽 2시에 검색창에 적었던 그 말…" |
| **② 명시적 출처** | 모든 통계·주장에 학술/공공 출처 표기. 본문 인용 + 하단 박스 + 각주. | "통계청 2023년 사회조사에 따르면…¹" |
| **③ 보편 현상으로 추상화** | 개인 경험을 "한국 30대의 공통 패턴"으로 확장. | "혼자만의 문제가 아닙니다. 30~49세 응답자의 47%가…" |

## 2. 5단 구조 표준 (Standard 5-Step Article Structure)

모든 글은 다음 5단 구조를 따른다. 길이는 글마다 조절 가능하나 **순서는 고정**.

```
┌─────────────────────────────────────────────────────┐
│ 1단. 후크 (Hook)            - 200~400자               │
│   독자가 자기 일로 받아들이는 한 줄·한 장면           │
│                                                       │
│ 2단. 보편 현상 (Universal)   - 400~700자              │
│   "당신만의 문제가 아닙니다" + 통계 1~2개             │
│                                                       │
│ 3단. 연구·데이터 (Evidence)  - 1,000~2,000자          │
│   학술 출처 2~4개 + 인용 + 해석                       │
│                                                       │
│ 4단. 통찰 (Insight)         - 500~1,000자             │
│   "그래서 우리에게 필요한 건…" 구조적 결론           │
│                                                       │
│ 5단. 자연스러운 다음 단계    - 200~400자              │
│   (Soft CTA) 강요 없는 다음 행동 안내                 │
└─────────────────────────────────────────────────────┘
```

### 각 단계별 체크포인트

**1단. 후크**
- 첫 문장은 의문문 또는 장면 묘사.
- "당신이 ○○한 적 있나요?" / "새벽 2시, ○○했던 사람이라면…"
- 광고적 표현 금지 ("놀라운 비밀!", "절대 안 알려주는…").

**2단. 보편 현상**
- 통계 1~2개 인용으로 "혼자가 아니다"를 증명.
- 한국 데이터 우선 (통계청, 보건복지부, KDI, 한국갤럽).
- 인용 후 "이건 개인의 문제가 아니다"라는 명제로 연결.

**3단. 연구·데이터**
- 학술 논문 2~4개 인용. 본문에 저자·연도 명시.
- 인용 형식: `프랭클(1946)은 ○○라고 말했습니다.`
- 인용문은 `*기울임체*` 또는 인용 블록(`> `)으로 처리.
- 해석은 어렵지 않게 — 일반 독자(20~50대) 기준.

**4단. 통찰**
- "그래서 우리에게 필요한 건"으로 시작 가능.
- 핵심 메시지 1개를 명확히 — 여러 결론을 늘어놓지 않음.
- 다이어리·상품 언급 금지. 추상적 원리만.

**5단. 자연스러운 다음 단계 (Soft CTA)**
- "오늘 밤, 이 한 질문만 적어보세요" 같은 즉시 행동.
- 상품 언급은 마지막 1~2문장에 **선택적**으로. (예: "더 깊은 진단이 궁금하다면 [무료 인생포트폴리오 진단]을…")
- 강요·재촉·할인 어휘 금지.

## 3. 블랙리스트 (Blacklist) — 본문 등장 금지 용어

> 이유: 내부 전략 문서의 비즈니스/마케팅 어휘가 독자용 콘텐츠로 새어 나오면 "광고"로 인식됨.

### A. 비즈니스/재무 지표 (Business Metrics)
```
CAC, LTV, ROAS, ARPU, MRR, BEP, 손익분기, 마진율, 펀딩,
시장 규모, TAM/SAM/SOM, 침투율, 전환율, 객단가, 매출, 영업이익
```

### B. 마케팅 전문 용어 (Marketing Jargon)
```
퍼소나, Persona, JTBD, 페인포인트, 니즈, 타깃, 세그먼트,
퍼널, 인게이지먼트, 리텐션, 어트리뷰션, 그로스 해킹, 락인,
업셀, 크로스셀, AARRR, 코호트, 옴니채널
```

### C. 전략 프레임 (Strategy Frameworks)
```
5중 해자, 5 Forces, SWOT, PEST, 4P, STP, BCG 매트릭스,
밸류체인, 블루오션, 레드오션, 파괴적 혁신, 카테고리 디자인
```

### D. 자기 검열용 카피 (Hype Copy)
```
대박, 무조건, 단언컨대, 최고의, 유일한, 비밀, 비법, 노하우 폭로,
○○억 매출, 인생역전, 절대, 100% 보장, 후회 없는 선택
```

### E. 내부 코드네임 (Internal Codes)
```
Meaning-Liver, P1/P2/P3/P4, M1~M5 해자, S1~S7 시나리오,
Pillar 001, Cluster A/B/C/D, 1+11 모델
```

## 4. 화이트리스트 (Whitelist) — 권장 어휘

### A. 독자 호명
```
당신 / 우리 / 사람들 / 30대의 우리 / 한국에서 살아가는 우리
```

### B. 학술 인용 패턴
```
"○○(연도)는 ○○라고 말했습니다."
"○○ 대학 연구팀의 ○○년 연구에 따르면…"
"○○ 학회지에 발표된 한 메타분석은…"
"통계청 ○○년 사회조사에 따르면…"
```

### C. 보편 현상 호명
```
"이건 당신만의 문제가 아닙니다."
"실제로 많은 사람이…"
"우리가 자주 마주치는 장면입니다."
"이런 갈증은 한국에서만의 현상이 아닙니다."
```

### D. 통찰 연결어
```
"그래서 우리에게 필요한 건"
"여기서 드러나는 것은"
"이 연구가 우리에게 알려주는 사실은"
"종합하면"
```

### E. Soft CTA 어휘
```
"오늘 밤" / "이번 주" / "한 줄만" / "딱 한 가지만"
"천천히" / "당신의 속도로" / "강요 없이"
```

## 5. 출처 인용 3중 체계 (Triple Citation System)

모든 학술·통계 데이터는 **세 곳에 동시 표기**한다. 정직함의 기본기.

### ① 본문 인라인 인용 (Inline Citation)
```markdown
프랭클(1946)은 *죽음의 수용소에서*에서 인간이 가장 견디기 어려운 것은
고통이 아니라 **의미 없는 고통**이라고 말했습니다.¹
```

### ② 하단 출처 박스 (Source Box at the bottom of article)
```html
<aside class="source-box">
  <h3>참고 자료</h3>
  <ol>
    <li>Frankl, V. E. (1946). <em>Man's Search for Meaning</em>. Beacon Press.</li>
    <li>통계청 (2023). <em>한국의 사회지표 2023</em>. kostat.go.kr</li>
  </ol>
</aside>
```

### ③ 본문 각주 번호 (Footnote Numbers)
```
…답한 비율은 34.7%입니다.¹
```
- 위첨자 숫자(¹²³…)로 본문과 하단 박스를 연결.
- 한 글에 평균 **3~6개 각주**.

## 6. 3단계 검증 프로세스 (3-Check Verification)

발행 전 다음 3단계를 거친다.

### Check 1. 블랙리스트 스캔
```bash
# 본문에 등장하면 안 되는 25개 용어 확인
grep -Ei "(CAC|LTV|ROAS|페르소나|JTBD|5중 해자|Meaning-Liver|P[0-9]|M[0-9])" public-draft.md
```
- 결과가 **0건**이어야 통과.
- 단, 인용 박스 내 학술명 등은 예외로 처리 (수동 검토).

### Check 2. 출처 매칭 검증
- 본문 각주 개수 = 하단 출처 박스 항목 수.
- 각 출처는 **저자/연도/제목/출처(저널·출판사·기관)** 4요소를 반드시 포함.
- 가능한 한 **DOI 또는 공식 URL** 첨부.

### Check 3. "3 No" 자가 점검 질문
발행 직전 다음 3가지를 점검:

1. **저자(나)의 이야기가 들어 있는가?** → 있으면 삭제.
2. **브랜드명이 본문(CTA 외)에 나오는가?** → 있으면 삭제.
3. **특정 인플루언서·경쟁사명이 나오는가?** → 있으면 삭제.

세 질문 모두 "아니오"여야 발행.

## 7. 2-Track 문서 시스템 (2-Track Document System)

각 글마다 **두 개의 문서**를 짝지어 작성한다. 물리적 분리로 누수 차단.

```
docs/strategy/diary/blog-posts/
├── 001-meaning-pillar/
│   ├── internal-brief.md   ← 내부용 (퍼소나, 키워드, SEO, 비즈니스 맥락)
│   └── public-draft.md     ← 독자용 (3 No 3 Yes 적용, 발행 직전 문안)
├── 002-assessment-drawer/
│   ├── internal-brief.md
│   └── public-draft.md
...
```

### internal-brief.md 표준 섹션
- 글 번호 / 제목 / 발행일
- 타깃 퍼소나 (P1~P4)
- 핵심 메시지 1줄
- SEO 키워드 (메인 + 롱테일 3~5개)
- 인용 예정 출처 리스트
- 본 블로그 페일러 (Pillar/Cluster) 위치
- 다음 글로의 내적 링크(Internal link) 안내
- CTA 강도 (Soft / Medium / 본 블로그는 Strong 사용 금지)

### public-draft.md 표준 섹션
- 발행 제목(H1)
- 후크 (200~400자)
- 보편 현상 (400~700자)
- 연구·데이터 (1,000~2,000자)
- 통찰 (500~1,000자)
- 다음 단계 (200~400자)
- 참고 자료 (Source Box)

## 8. SEO·메타데이터 표준

발행 HTML은 다음을 **모두** 포함한다:

- `<title>` (60자 이내) / `<meta name="description">` (155자 이내)
- `<link rel="canonical">` 절대 URL
- Open Graph (`og:title`, `og:description`, `og:url`, `og:type=article`, `og:image`)
- Twitter Card (`twitter:card=summary_large_image`)
- `<link rel="alternate" hreflang="ko">` + `hreflang="en"` + `hreflang="x-default"`
- JSON-LD `Article` 스키마 (헤드라인, 저자, 발행일, 수정일, 메인이미지)
- `<article>` 시맨틱 태그 + `<h1>` 1개 + `<h2>` 본문 분리

(기존 `blog/posts/2026-05-19-30s-career-shaking-first-check.html` 패턴을 표준 템플릿으로 사용)

## 10. 부록: 빠른 자가 점검 5문항

발행 직전, 다음 질문에 모두 "예"여야 합니다.

1. ☑ 후크가 "당신/우리"로 시작하나요?
2. ☑ 본문에 학술/공공 출처가 3개 이상 명시되었나요?
3. ☑ 블랙리스트 25개 용어를 스캔하여 **0건**임을 확인했나요?
4. ☑ "3 No(저자/브랜드/인플루언서)" 검증을 통과했나요?
5. ☑ 마지막 CTA가 강요 없는 Soft CTA인가요?

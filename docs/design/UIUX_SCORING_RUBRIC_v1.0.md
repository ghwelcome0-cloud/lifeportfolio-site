# 인생포트폴리오 UX/UI 톤앤매너 평가 기준 (Scoring Rubric) v1.0

> **목적**: 홈페이지(PC·모바일, 한/영)의 톤앤매너 품질을 객관적·반복적으로 측정하는 영구 기준.
> 매 개선 라운드마다 이 표로 재채점하여 **상향·축적**을 보장한다.
>
> **최초 작성**: 2026-06-25
> **기준 모델**: Linear · Stripe · Notion · Framer 제작 상위 랜딩 페이지가 공유하는 6대 원칙
> **목표 점수**: 종합 **9.0 / 10 이상**

## 1. 평가 6대 항목 (각 10점, 가중치 동일)

| # | 항목 | 핵심 질문 | 만점(10) 기준 |
|---|------|-----------|--------------|
| **A** | **디자인 토큰 체계성** | 컬러·타이포·간격이 토큰화되어 일관되는가? | `:root`에 color/type/space 토큰 완비, 하드코딩 색상 ≤ 5종, 토큰 우회 없음 |
| **B** | **타이포그래피 스케일** | 명확한 모듈러 스케일을 쓰는가? | 폰트 사이즈 6~8단계로 통일, 0.5px 난립 0, 본문 ≥15px |
| **C** | **시각적 위계 (3초 룰)** | 첫 화면에서 3초 안에 가치가 전달되는가? | 히어로 = 헤드라인 1 + 서브 1 + CTA 1 중심, 시선 동선 단일 |
| **D** | **여백 리듬** | 8px 그리드로 호흡 있는 간격을 쓰는가? | margin/padding이 8의 배수(8·16·24·32·48·64)로 정렬, 과감한 여백 |
| **E** | **정보 밀도** | 스캔 가능한가? (읽지 않고 훑어도 이해) | 한 화면당 1메시지, 짧은 라벨·숫자·아이콘 위주, 서술형 최소 |
| **F** | **성능·기술 품질** | 빠르고 콘솔이 클린한가? | LCP < 2.5s, 콘솔 에러 0, 캐싱 가능한 구조 |

## 4. 측정 방법 (재현 가능하도록)

```bash
# B. 폰트 사이즈 종류 (적을수록 좋음, 목표 ≤8종)
grep -oE "font-size:[^;}]*" index.html | sort | uniq -c | sort -rn

# A. 하드코딩 색상 종류 (목표 ≤5종 + 토큰)
grep -oE "#[0-9a-fA-F]{6}|#[0-9a-fA-F]{3}\b" index.html | sort | uniq -c | sort -rn

# D. margin/padding 값 분포 (8의 배수 정렬도 확인)
grep -oE "(margin|padding)[^:]*:[^;}]*" index.html | grep -oE "[0-9]+px" | sort | uniq -c | sort -rn

# F. 로드 시간 / 콘솔 — PlaywrightConsoleCapture 또는 Lighthouse
```

**라이브 검증 필수 규칙**
- `cleanUrls:true` → 반드시 클린 URL로 확인 (`/report` O, `/report.html` ✗ 301)
- 한/영 양쪽, PC/모바일 양쪽 모두 정상 작동 확인 후 점수 인정

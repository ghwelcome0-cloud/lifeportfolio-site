# PART 1. 자동 버전 (AI 엔진 기반)

## 1-1. 시스템 흐름

```
사용자 응답(report.html 폼)
  → RTDB responses/{uid}/{sid}
  → report-engine.js (1차 합성)
  → report-engine-v4.js (품질 업그레이드)
  → reports/{uid}/{sid} 저장
  → 사용자 "🚀 맞춤형 실행 프로그램 생성하기" 클릭
  → program-loading.html 4단계 파이프라인
  → program-engine.js (ProgramEngine.build)
  → programs/{uid}/{sid} 저장
  → MyPage 노출 / PDF 인쇄
```

## 1-2. 자동 제작 표준 절차 (≈ 5분)

| 단계 | 행위자 | 작업 | 검증 |
|---|---|---|---|
| A1 | 회원 | 진단 검사 응답 제출 | 80컬럼 모두 입력 |
| A2 | 시스템 | report-engine 자동 합성 | toneKey·primaryDomain 도출 로그 |
| A3 | 시스템 | report-engine-v4 품질 업그레이드 | C1~C12 자동 검증 |

| A5 | 회원 | "🚀 맞춤형 실행 프로그램 생성하기" 클릭 | mobile/PC 1.5s 이내 페이지 전환 |
| A6 | 시스템 | program-loading.html 4-stage 파이프라인 | Stage 1~4 모두 완료 |

# PART 2. 수동 버전 (운영자 직접 합성)

운영자는 **응답 데이터 + 본 매뉴얼만**으로 김영식 모범 사례와 동일한 수준의 결과물을 만들 수 있어야 한다. 코드 작성 불필요. 두 가지 방법 중 선택.

## 2-A. 방법 A — AI 일괄 처리 (권장, ≈ 25분)

| 단계 | 시간 | 작업 |
|---|---|---|
| ① 입력 묶음 준비 | 5분 | Excel `responses` 시트에서 1행 추출 → 80컬럼을 JSON으로 정리 |
| ② AI 프롬프트 송부 | 10분 | 본 매뉴얼 PART 0 + 골든 JSON + 응답 JSON을 함께 첨부 → AI에게 "김영식 모범 사례와 동일한 7섹션 리포트 + 7섹션 프로그램을 합성해 주세요" 요청 |
| ③ 응답 검수 | 5분 | PART 5 100점 체크리스트로 채점 |
| ④ PDF 생성·발송 | 5분 | Word/PPT 템플릿에 텍스트 붙여넣기 → PDF 변환 → 이메일 |

**AI 프롬프트 표준**:
```
[입력] 회원 응답 JSON (80컬럼)
[규칙] PRODUCTION_MANUAL_v2.0.md PART 0의 매핑 알고리즘
[모범] KYS_GOLDEN_REFERENCE.json 의 reportOutput / programOutput 구조
[출력 형식] JSON {report: {...7섹션}, program: {...7섹션}}
[자연성 규칙] R6 — '사람을 ~으로 ~하는 사람' 패턴 → '~으로 사람을 ~ 내는 사람'
[고유성 규칙] R4 — 응답 중 1개라도 다르면 산출물 1개 이상 변경
```

## 2-B. 방법 B — 단계별 수기 처리 (정밀, 1~2시간)

5단계 M1~M5 워크플로우. 코드 작성 없이 운영자가 응답을 보면서 직접 텍스트를 합성한다.

### M1. 응답 데이터 추출 (15분)

1. Spreadsheet `responses` 시트에서 회원 행을 이메일·타임스탬프로 검색
2. 핵심 슬롯 11개를 메모장에 옮김:
   - Q1 (잘 맞는 활동), Q3 (TOP3 강점), Q13 (핵심 가치 3), Q41 (열정 영역), Q47 (성장 포인트), Q49 (몰입 환경), Q63 (컴퍼스), Q73 (가치 결합), Q75 (1·2 도메인), 성취 조건, 이름·이메일

### M2. 매핑 도출 (10분)

| 도출 항목 | 방법 |
|---|---|
| `compassKw` | Q63 그대로 |
| `primaryDomain` | Q75[0] → 13영역 정규화 (부록 A 참조) |
| `secondaryDomain` | Q75[1] |
| `subType` | Q1·Q3·Q13·Q41 키워드를 5타입에 손으로 채점, 최고점 |
| `toneKey` | Q13 가치 + Q63 컴퍼스 → PART 5 톤 표 참조 |
| `careers` | 부록 A 매핑표에서 `primaryDomain × subType` 셀의 직업 1개 + 융합형 + Q41 |
| `education` | 부록 B에서 같은 영역의 단·중·장 각 1 |

### M3. Word 리포트 합성 (30분)

`docs/manual/template_report.docx` 12-section 템플릿 사용. PART 3 김영식 모범 사례를 보고 같은 구조로 텍스트만 갈아끼움. **편집 가능 슬롯 5개**(사명·비전 한 줄 / 3주 루틴 본문 / 신규 직업 리스트 / 리스크 완화 / 마무리 메시지)만 자유롭게 다듬을 수 있고 나머지 슬롯은 자동 계산값을 따른다.

### M4. PowerPoint 프로그램 합성 (30분)

`docs/manual/template_program.pptx` (3주/3개월/1년 슬라이드). PART 3 김영식 모범 프로그램을 보고 같은 7섹션을 채움.

### M5. PDF 발행 + 이메일 (15분)

- 파일명 규칙:
  - `최신_인생포트폴리오_{성명}_{YYYY-MM-DD}.pdf`
  - `최신_{성명}님_맞춤_실행_프로그램_{YYYY_MM-DD}.pdf`
- RTDB 마킹: `programs/{uid}/{sid}.manualProgramStatus = "manual"`
- 이메일 발송 + 회신 대기

- **짧은 한 줄**: 자기 자리에 있는 사람이 사람을 원칙으로 지킨다.
- **컨텍스트**: 경영과 종교의 자리에서, 원칙을 나침반 삼아.
- **다이어리**: 나는 어떤 원칙으로 살지를 분명히 하면서, 내가 선 자리에서 곁에 있는 사람들 곁에 (특히 돈과 자원이 흐르는 길목에서), 마음을 열어주는 따뜻한 사람이자 곁에 있어주는 사람으로 매일을 살아간다.
- **한 줄 통합본**: 당신의 사명은, 경영과 종교의 자리에서, (특히 돈과 자원이 흐르는 길목에서) 양심이 부르는 자리를 잃지 않으며, 사람의 곁을 머물게 하면서 자기 색대로 깊어져 가는 한 사람으로 살아가는 것입니다.

### ③ 비전 (Vision)
- **짧은 한 줄**: 원칙으로 사람을 지켜 내는 사람으로 기억된다.
- **컨텍스트**: 10년 뒤, 경영과 종교의 자리에서 원칙을 잃지 않은 사람으로.
- **10년 뒤 다이어리**: 10년 뒤 사람들은 나를 "마음을 열어주는 따뜻한 사람", "곁에 있어주는 사람", "한 원칙으로 평생을 산 사람"으로 기억한다.
- **한 줄 통합본**: 당신의 비전은, 경영과 종교의 자리에서 곁이 따뜻하면서도 자기 색대로 자라 가는 한 사람이자 자기 원칙이 또렷한 사람으로 자리잡는 것입니다.

### ④ 실행 프로파일 + TOP3 강점 + TOP2 성장
- **실행 유형**: 흐름을 짜는 사람
- **실행 스타일**: 양심이 부르는 자리에서 — 서두르지 않는 흐름으로, 운영체계로 단단히 · 말로 자리를 만들며
- **TOP3 강점**: 분석적 정직성 / 전략적 성취력 / 분석력
- **TOP2 성장**: 갈등 상황에서의 단단함 / 의견 단호한 표현

### ⑤ 추천 진로·교육·확장
- 진로: ① 경제 평론가 ② 경제·종교 융합형 영성 콘텐츠 크리에이터 ③ 핀테크 창업가
- 교육: ① 경제 다큐 워크숍 ② 영상 글쓰기 과정 ③ 경제 저널리즘 과정
- 확장: 경영을 본업·종교를 부업·연구로 / 큰 그림을 작은 실험으로 / 경영 통찰을 종교 콘텐츠로 변환

### ⑥ 4축 응답 강도
| 축 | 본질 | 한 마디 | 강도 |
|---|---|---|---|
| 자기이해 | 신중한 매개로 '나는 누구인가'를 끊임없이 다시 정의해 가는 힘 | 조용한 시간 속에서 자기 자신과 가장 깊이 대화하는 사람 | 92% |
| 자기표현 | 진정성을 매개로 자기 감정을 조심스럽게 드러내는 힘 | '들어주는 사람이 있다면 나도 말하고 싶다'고 느끼는 사람 | 85% |

## 3-4. 프로그램 7섹션 (실측 출력 그대로)

### ① 헤드라인 + 사명 인용
> 비전 중심의 창조형 실행가 · 큰 그림을 그려내고 작품으로 증명하는 창조의 사람
> "이 프로그램은 김영식님의 사명 — '자기 자리에 있는 사람이 사람을 원칙으로 지킨다' — 을 매일 한 호흡으로 옮깁니다."

### ② 분기 테마
**사람의 원칙을 작품으로 펼치는 분기**
- 이미 김영식님은 사람의 원칙을 작품 결로 끌어안고 있습니다.
- 이번 분기는 그 원칙을 '사람 듣기 → 작품 펼치기 → 원칙 자산화'로 구조화하는 시간입니다.
- 경영에서 한 사람의 원칙이 작품 한 줄로 옮겨질 때 새 결이 열립니다.

### ③ 3주 실행 루틴

**1주차 — 아이디어를 밖으로 꺼내기**
- 매일 아침, 오늘 작품에 닿을 한 사람의 원칙을 한 줄 메모합니다.
- 대화 중 사람의 원칙 단어를 1개씩 받아 적어 작품 재료로 모읍니다.
- 하루 끝, 사람의 원칙이 작품에 스민 한 장면을 3줄로 기록합니다.
- 관심 활동 중 1개를 이번 주에 자기 결로 시도합니다.
- *효과*: 아이디어 외화 · 콘셉트 좁히기 · 레퍼런스 정렬 · 착수 가속

**2주차 — 프로토타입 빠르게 끝내기**
- 주 3회, 경영의 초안을 한 사람에게 보여 주고 한 문장 피드백을 받습니다.
- 내가 만난 사람들의 원칙을 작품 카피 1줄로 묶습니다.
- 사명 — '자기 자리에 있는 사람이 사람을 원칙으로 지킨다'을 함께 펼쳐 줄 3명에게 발행 초안을 공유합니다.
- 성취 조건('내가 의미 있다고 여긴 일을 마쳤을 때')을 이번 주 한 번 의식적으로 적용합니다.
- *효과*: 프로토타입 마감 · 피드백 수집 · 떨어내기 결정 · 발행 임박

**3주차 — 발행과 다음 비전 연결**
- 원칙이 살아나는 한 사람과 30분 깊이 대화를 진행합니다.
- 지난 3주 사람에게서 길어 올린 원칙을 5줄 회고로 묶습니다.
- '원칙으로 사람을 지켜 내는 사람'을 향한 분기 작품을 사람 결에서 명문화합니다.
- 몰입 환경(정돈된 실내·몰입/휴식 분리)을 한 번 의도적으로 만들어 깊이 한 호흡을 갖춥니다.
- *효과*: 외부 발행 1건 · 반응 데이터 확보 · 다음 비전 한 줄 · 발행 자산화

### ④ 3개월 마일스톤 (KPI 4)

| 목표 | KPI |
|---|---|
| 사람과 함께 펼치는 원칙 정착 | 주 3회 사람의 원칙 단어 1개 수집 |
| 원칙 작품 가시화 | 경영 발행 5건 (사람 카피) |
| 원칙을 펼친 작품 자산화 | 3개월 사람 결 발행 12건 |
| 관심 주제 결과 자리 잡기 | 분기 말 결과 3건 가시화 |

### ⑤ 1년 확장 비전
> 1년 뒤, 경영과 종교의 자리에서 '원칙으로 사람을 지켜 내는 사람'으로 자리잡게 됩니다. 원칙이 곧 작품이 되는 자기만의 발행 구조를 갖추게 됩니다.
- 분기마다 1개 이상의 외부 발행물 / 결과를 보유
- 사명 한 문장으로 자기 설명 가능 수준 도달
- 원칙을 매개로 연결된 동료 / 팬 5명 이상 확보
- 관심 활동을 시그니처 결과로 매듭짓기

### ⑥ 부스터 3 + 리스크 3

**부스터**
- 강점 활용 — 아이디어 캡처 루틴 (아이디어 캡처 노트 · 레퍼런스 보드 · 1주 프로토타입 시트)
- 보완 훈련 — 프로토타입 스프린트 (마감 보드 · 피드백 노트 · 떨어내기 체크리스트)
- 핵심 전략 — 비전 카피 점검 세션 (발행 채널 · 카피 시트 · 분기 작품 인덱스)

**리스크**
| 리스크 | 완화 |
|---|---|
| 아이디어가 많아 발행이 늦어질 수 있음 | '이번 주 1개'에만 아이디어 보관 |
| 반응이 약하면 동력이 빠르게 떨어짐 | 발행 자체를 성과로, 반응은 학습 데이터로 |
| 마무리보다 새 시작에 끌릴 수 있음 | 현재 작품 발행 전 새 프로젝트 보류 |

### ⑦ 마무리 메시지
> 이 프로그램은 더 많은 아이디어를 만들기 위한 설계가 아닙니다. 이미 김영식님 안에 자리 잡은 사명 — '자기 자리에 있는 사람이 사람을 원칙으로 지킨다' — 이 세상이 맞닿을 수 있는 작품으로 매일 이어지도록 빚는 구조입니다. 오늘의 원칙 한 줄 메모부터 시작하면 충분합니다.

# 개발자 보완 요청서 — 젠스파크 제작 워크플로 재설계 (길 A)

> **작성 목적:** 현재 스튜디오 사이트의 활용도를 끌어올리기 위한 보완 요청.
> **핵심 전략 = 길 A "사이트는 사령탑, 젠스파크는 공장".**
> 사이트는 시장조사 결과·대본·완성도 게이지·전환 스코어카드·발행 대기열을 **저장/관리/추적**한다.
> 무거운 AI 생성(시장조사·대본·이미지·영상·숏폼)은 **젠스파크 플랫폼 안에서** 수행한다.
> 연결고리 = 각 단계마다 **"이 작업용 프롬프트 [복사]" 버튼 → 젠스파크에 붙여넣기 → 결과를 사이트에 다시 저장**.

---

## 0. 반드시 먼저 알아야 할 전제 (검증 완료)

- 젠스파크는 **공개 개발자 API가 없다.** 유료 회원이어도 외부 앱(이 사이트 포함)이 코드로 젠스파크 크레딧을 호출해 자동 생성하는 것은 **불가능**하다. (출처: r/genspark_ai API availability, Sacra "Genspark API for SaaS Embedding", 젠스파크 헬프센터)
- 그러나 **젠스파크 플랫폼 안에서 사람이 직접** 시장조사~숏폼 제작을 하는 것은 **가능**하다. 사용 도구:
  - **AI Sheets 2.0** — YouTube 검색·집계, 조회/좋아요/댓글 분석 대시보드, 웹스크래핑, 트렌드 감지 (출처: Genspark 공식 LinkedIn, r/aicuriosity, WindowsForum)
  - **Super Agent / AI Chat** — 대본·숏폼 스크립트·업로드 설정 생성 (2026 무제한)
  - **AI Image / AI Video** — 컷별 이미지·영상 (2026 이미지 무제한)
  - **AI Drive** — 내부 자료(블로그 44편·진단 설계) 업로드 후 근거로 활용
- 따라서 **이 사이트는 "AI를 호출하는 앱"이 아니라 "젠스파크 작업을 지시·수집·추적하는 운영 콘솔"로 만든다.** 자동화 기대(❌)를 버리고 **반자동 브리지(✅)**로 확정.

## 1. 공통 UI 패턴 — 모든 단계에 동일 적용

각 제작 단계 화면은 아래 **4블록 구조**를 가진다.

```
┌─────────────────────────────────────────────┐
│ ① 입력 요약 (이 콘텐츠의 제목·타깃·유형 등 자동 표시)  │
│ ② [젠스파크용 프롬프트] (자동 생성) ────── [📋 복사]    │
│ ③ "젠스파크에 붙여넣고 → 결과를 아래에 붙여넣으세요" 안내  │
│ ④ 결과 붙여넣기 칸(텍스트) / 파일 업로드 칸(이미지·영상)   │
│    → [저장] 누르면 해당 콘텐츠 객체에 반영               │
└─────────────────────────────────────────────┘
```

- ②번 프롬프트는 **콘텐츠 데이터(제목/타깃/유형/SEO 키워드/블로그 근거)를 자동 치환**해서 생성. 사용자가 빈칸 채울 필요 없음.
- 모든 프롬프트 끝에 **브랜드 톤·금지 규칙·전환 목표를 자동 주입** (아래 5번 가치관 가드 연동).
- 복사 버튼은 한 번 누르면 "복사됨 ✓" 토스트. **이것이 사이트 활용도의 핵심.**

## 2. 5단계 워크플로 + 검증된 프롬프트 템플릿

### 단계 ① 시장조사 · 빅데이터 분석 (→ `bigdata`, `scorecard.reach`)

**도구:** 젠스파크 **AI Sheets 2.0** (YouTube 집계 지원)
**사이트 역할:** 키워드 리스트 관리 → 프롬프트 생성 → 결과(검색량·경쟁·예상조회) 붙여넣기 저장 → `기회 점수` 계산

**[복사용 프롬프트 — AI Sheets용]**
```
너는 유튜브 채널 시장분석가다. 아래 키워드들에 대해 표로 정리하라.
[채널] 인생포트폴리오 (인생설계·자기경영·진로·사명·강점 진단)
[타깃] 진로·방향·의미를 고민하는 20~40대
[키워드 목록] {사이트가 bigdata 키워드 자동 삽입}
[각 행에 채울 열]
- 키워드
- 월간 검색 추정(높음/보통/낮음 + 근거)
- 상위 영상 평균 조회수
- 경쟁 강도(0~100)
- 콘텐츠 공백(아직 안 다뤄진 각도)
- 인생포트폴리오 진단 연결 적합도(높음/보통/낮음)
[작업] 유튜브에서 각 키워드를 검색해 상위 영상의 조회·댓글 경향을 집계하고,
"기회 점수 = (검색수요 높음 + 경쟁 낮음 + 진단 연결 높음)"이 큰 순으로 정렬하라.
[출력] 표 + 추천 TOP 3 키워드와 그 이유.
```

**사이트 저장 매핑:** 결과 표를 붙여넣으면 → 각 키워드의 `searchVolume / competition / estViews`에 반영, `scorecard.reach.estViews` 갱신, `기회 점수`로 다음 제작 우선순위 자동 추천.

### 단계 ② 대본 제작 (→ `script`, 5섹션) — **내부 자료 + 시장조사로 효과성 검증 후 제작**

**도구:** 젠스파크 **Super Agent / AI Chat** + **AI Drive**(블로그 44편 업로드)
**사이트 역할:** 선택한 주제 + ①의 시장조사 결과 + 출처 블로그를 묶어 프롬프트 생성 → 5섹션 대본 붙여넣기 저장 → **가치관 가드 자동 점검**(5번)

**[복사용 프롬프트 — Super Agent용]**
```
너는 "인생포트폴리오" 유튜브 채널의 전문 카피라이터다.
[브랜드 톤] 따뜻하지만 핵심을 찌르는 멘토. 사탕발림 없이 정직하게. 과장된 동기부여 금지.
[목표] 시청자를 lifeportfolio.co.kr의 15분 76문항 진단검사로 유입 → 첫 주 설계도(9,900원) 전환.
[이 영상 주제] {제목/타깃/유형 자동 삽입}
[시장 근거] {① 시장조사 TOP 키워드·공백 자동 삽입}
[내부 근거 자료] (AI Drive에 올린 블로그) {출처 블로그 제목·핵심 논지·학술근거 자동 삽입}
[작성 규칙 — 구매심리 6요소(성경적 재정의)를 5섹션에 배치]
1) 후킹/문제인식 (0:00~0:40): 시청자의 진짜 고민을 정직하게 짚는다. 공포·결핍 자극 금지.
2) 공감 (~2:30): 정죄 없이 함께 선다. "당신이 틀렸다"가 아니라 "그럴 수 있다".
3) 가치/권위 (~본론): 근거(블로그 학술 자료)로 신뢰를 준다. 과장·단정 금지.
4) 증거 (~후반): 통계·연구·사례를 구체적으로 제시.
5) CTA/욕구→행동: 압박 없이 "초청". "지금 안 하면 손해" 금지 → "관심 있으시면 편하게".
[금지] 공포(망한다/늦는다), 결핍(당신은 부족하다), 분노·편가르기(그들이 속였다),
조작(사실은~비밀), 탐욕(쉽게 떼돈), 압박(지금 당장/마지막 기회).
[출력] 위 5섹션을 [후킹]/[공감]/[가치]/[증거]/[CTA] 머리표로 구분해 작성.
각 섹션 끝에 (예상 길이/초) 표기. 마지막에 영상 설명·해시태그·고정 댓글 초안 포함.
```

**사이트 저장 매핑:** 붙여넣기 → `script.hook/empathy/value/proof/cta` 자동 분할 저장 + `script.raw` 보관. 저장 즉시 **가치관 가드(5번) 자동 실행 → 가치관 안전도 게이지 표시.**

### 단계 ③ 음성 제작 (→ `video.voiceBlobId`) — 대표님이 직접 녹음

**도구:** 대표님 직접 녹음 (또는 보조로 젠스파크 TTS)
**사이트 역할:** 대본을 **녹음용 큐시트**로 보여주기(섹션별 줄바꿈·호흡 표시) → 녹음한 음성 파일 **업로드** → `video.voiceBlobId`에 Blob 저장 → 길이(초) 자동 측정해 타임라인 기준값으로.

- 추가 버튼: **[대본 → 녹음 큐시트 변환]** (긴 문장 자동 분절, 강조 표시).
- 음성 파일은 IndexedDB Blob 저장 + JSON Export 시 base64 포함(기존 백업 규칙 유지).

### 단계 ④ 컷별 이미지 / 영상 제작 (→ `thumbnail`, `video.scenes/clipBlobIds`)

**도구:** 젠스파크 **AI Image(무제한)** + **AI Video**
**사이트 역할:** 대본을 **컷(scene) 단위로 분해** → 컷마다 이미지/영상 프롬프트 자동 생성 → 생성물 업로드 → 타임라인에 배치

**[복사용 프롬프트 — 컷별 이미지/영상용]**
```
너는 유튜브 영상 비주얼 디렉터다. 아래 대본을 컷 단위로 나누고 각 컷의 이미지를 생성하라.
[채널 톤] 딥네이비(#0E2A47)·그린(#1FA36B) 기반, 따뜻하고 신뢰감 있는 미니멀. 자극적·공포 연출 금지.
[대본] {섹션별 대본 자동 삽입}
[작업]
1. 대본을 3~6초 단위 컷으로 분해(컷 번호 + 해당 내레이션 + 화면 설명).
2. 각 컷마다 이미지 생성 프롬프트(영문)를 작성. 인물은 webtoon 스타일 일관 유지.
3. 썸네일용 1장은 별도로: 큰 한글 텍스트 자리 + 시선 끄는 구도(클릭베이트 과장 금지).
[출력] 컷 표(번호/시간/내레이션/화면/영문 프롬프트) + 썸네일 프롬프트.
```

**사이트 저장 매핑:** 컷 표를 붙여넣으면 → `video.scenes[]`(번호·시간·내레이션·화면설명·프롬프트) 자동 생성. 컷별 이미지/영상 업로드 → `video.clipBlobIds` / `thumbnail.blobIds`. 썸네일 선택 → `thumbnail.chosenBlobId`.

### 단계 ⑤ 업로드 설정 + 롱폼 → 숏폼 변환 (→ `description/hashtags/pinnedComment`, 신규 숏폼 콘텐츠)

**도구:** 젠스파크 **Super Agent**
**사이트 역할:** 업로드 메타 자동 생성 + **롱폼 대본에서 숏폼 후보 자동 추출 → 새 `contentType:"pulling"` 콘텐츠로 복제 생성**

**[복사용 프롬프트 A — 업로드 설정]**
```
너는 유튜브 SEO 담당자다. 아래 영상의 업로드 설정을 작성하라.
[제목] {제목} [대본 요약] {대본 자동 삽입} [진단 링크] lifeportfolio.co.kr
[작업] 1) 클릭을 유도하되 과장 없는 제목 후보 3개
2) 설명문(첫 2줄에 핵심+진단 링크, 이후 챕터 타임스탬프, 해시태그)
3) 해시태그 8~12개(검색·연관 혼합)
4) 고정 댓글(진단 초청 + 다른 키 콘텐츠 핑)
[금지] 공포·압박형 표현. 초청형 톤 유지.
```

**[복사용 프롬프트 B — 롱폼 → 숏폼]**
```
너는 숏폼 편집 기획자다. 아래 롱폼 대본에서 30~60초 숏폼 3개를 뽑아라.
[롱폼 대본] {대본 자동 삽입}
[작업] 각 숏폼: (1) 첫 2초 후킹 문장 (2) 핵심 한 가지 (3) "전체 영상/진단" 유도 한 줄.
세로 9:16 화면 컷 구성 포함. 통념 반전·공감 미끼 우선. 공포·낚시 금지.
[출력] 숏폼별 [후킹]/[본론]/[CTA] + 컷 설명.
```

**사이트 저장 매핑:**
- A 결과 → `description / hashtags / pinnedComment` 저장.
- B 결과 → **각 숏폼을 새 콘텐츠 객체(`format:"short"`, `contentType:"pulling"`, `funnel:"awareness"`)로 자동 생성**하고 원본 롱폼과 `sourceRefs`로 연결. 발행 대기열에 자동 등록.

## 3. 워크플로를 데이터 모델에 반영 (개발자 작업 항목)

기존 백업 스키마(`barrel` 11요소, `script` 5섹션, `scorecard` 4단계, `metrics` 4지표)는 **유지**. 아래만 보완:

1. 콘텐츠 객체에 **`workflowStage`** 추가(`market→script→voice→visual→upload→short` 6단계). 각 단계 완료 여부 체크 → 완성도 게이지(`barrel`)와 연동.
2. `video.scenes[]` 스키마 명세화: `{ no, startSec, endSec, narration, screen, imagePrompt, blobId }`.
3. 숏폼 파생 콘텐츠는 `sourceRefs`에 원본 id 기록(롱폼↔숏폼 추적).
4. 각 단계의 "젠스파크 프롬프트"는 **템플릿 + 콘텐츠 데이터 치환**으로 런타임 생성(하드코딩 금지).

## 4. 백업 검토에서 발견된 누락 — `scorecard.psychology` 성경적 6요소 필드 (필수)

**문제:** 업로드된 백업의 `scorecard.psychology`가 **빈 객체 `{}`**. 우리 채널의 핵심 차별점인 성경적 6요소가 데이터로 안 들어가 있음.

**요청:** `scorecard.psychology`를 아래 구조로 정의하고, 대본 작성 화면에서 각 요소를 0~100으로 자가평가/체크하게 한다.

```json
"psychology": {
  "problemAwareness": { "score": 0, "note": "" },   // 문제인식 — 시청자의 진짜 고민을 정직하게 짚었나 (공포 자극 X)
  "empathy":          { "score": 0, "note": "" },   // 공감 — 정죄 없이 함께 섰나
  "authority":        { "score": 0, "note": "" },   // 권위 — 근거(자료)로 신뢰를 줬나 (과장 X)
  "evidence":         { "score": 0, "note": "" },   // 증거 — 통계·연구·사례가 구체적인가
  "desire":           { "score": 0, "note": "" },   // 욕구 — 더 알고 싶게 했나 (조장 X, 초청 O)
  "action":           { "score": 0, "note": "" }    // 행동 — 압박 없이 진단으로 초청했나
}
```

**각 요소의 성경적 재정의(화면 툴팁으로 표시):**

| 요소 | 일반 마케팅(지양) | 성경적 재정의(지향) | 근거 |
|---|---|---|---|
| 문제인식 | 공포·위기감 조성 | 정직한 진단, 진짜 필요를 비춤 | 빌4:8 참된 것 |
| 공감 | 동정 유발 후 이용 | 함께 섬, 정죄 없음 | 빌2:3-4 섬김 |
| 권위 | 권위로 압도 | 근거로 신뢰, 청지기의 책임 | 마24:45 충성된 청지기 |
| 증거 | 과장·체리피킹 | 검증된 사실로 진실되게 | 빌4:8 |
| 욕구 | 결핍·탐욕 자극 | 선한 갈망으로 초청 | 빌4:8 사랑받을 만한 것 |
| 행동 | 압박·낚시 | 자유로운 초청, 강요 없음 | 빌2:3-4 |

> 핵심 성구 3종: **마태복음 24:45**(사명·청지기) / **빌립보서 4:8**(진실성) / **빌립보서 2:3-4**(섬김).
> 막시즘의 분노·헤겔의 정반합 대립이 아니라, **사랑·진실·섬김 기반**.

**완성도 게이지 연동:** 6요소 평균 점수를 `barrel`의 `script` 항목 보조 지표로 표시(별도 "구매심리 충실도" 미니 게이지).

## 5. 가치관 가드(금지 표현 점검) 연동 — 대본 저장 시 자동 실행 (필수)

**현황:** 금지 표현 사전(`values-guard-checklist.md`)은 작성됐으나, 백업 스키마에 **연동 흔적이 없음**.

**요청:** 단계 ②(대본 붙여넣기) 직후 **자동으로 금지 표현 매칭**을 실행한다.

- 동작: 키워드/패턴 매칭(무거운 AI 판정 X). 6개 카테고리(공포/결핍/분노/조작/탐욕/압박).
- 의사코드:
  ```
  감지수 = count(대본 ∩ 금지패턴사전)
  가치관_안전도 = 100 - (감지수 × 가중치)   // 강제 차단 아님, 사람이 최종 판단
  if 감지 → 해당 문장 노란색 하이라이트 + ✅ 대안 표현 제안
  ```
- 데이터 저장: 콘텐츠 객체에 `valuesGuard: { score: 100, flags: [ { category, matched, suggestion, sentence } ] }` 추가.
- 경고 문구 예: `"이 표현은 사랑·진실 기준(빌4:8)에서 벗어날 수 있습니다. 아래 대안을 참고하세요."`
- 사전 원본은 `values-guard-checklist.md`를 그대로 규칙 소스로 사용(앱에 JSON으로 임베드 또는 fetch).

## 6. 우선순위 (개발자 권장 순서)

1. **(P0)** 공통 4블록 UI + "프롬프트 복사" 버튼 — 활용도 체감의 핵심.
2. **(P0)** 단계 ② 대본 + `scorecard.psychology` 6요소 + 가치관 가드 연동(4·5번).
3. **(P1)** 단계 ① 시장조사 AI Sheets 프롬프트 + bigdata 저장 매핑.
4. **(P1)** 단계 ④ 컷별 이미지/영상 + `video.scenes` 스키마.
5. **(P2)** 단계 ⑤ 업로드 설정 + 롱폼→숏폼 파생 생성.
6. **(P2)** 단계 ③ 녹음 큐시트 + 음성 업로드.

## 7. 정직한 한계 (대표님·개발자 공유)

- 젠스파크 작업은 **사람이 직접** 해야 한다(복사→붙여넣기→업로드). 완전 자동 아님.
- 시장조사 수치는 **추정·수동 집계**다. 정확한 검색량 API는 없음(유료 vidIQ/Vling 별도).
- 가치관 가드는 **키워드 매칭**이라 맥락 오탐 가능. 최종 판단은 사람.
- 이 한계를 받아들이는 대신, **비용 0(크레딧만) + 우리만의 전환·가치관 추적**을 얻는 구조다.

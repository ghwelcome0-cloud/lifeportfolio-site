## 0. 목표 (사용자 요구 verbatim 반영)

- 사용자가 고른 **관심 분야(Q75, N개)를 "나열"이 아니라 "융합"** 하여 **새로운 단 하나의 정체성**으로 표현한다.
  - ✅ 원함: `"신념을 가르쳐 조직으로 키우는 일"` (종교+교육+경영 → 원단어가 사라진 제4의 정체성)
  - ❌ 배제: `"종교·교육·경영이 만나는 자리"` (연결어 나열), `"종교와 교육, 경영을 잇는"` (분야 분리 노출)
- **(A) 규칙 기반 융합** 채택 — 분야를 **속성으로 분해 → 재조합**. (B) LLM 실시간 융합은 비용·비결정성 때문에 배제.
- **고차원 수학 원리 적용** → 고객이 **고유성·맞춤화**를 실제로 경험.
- **모든 고객·모든 조합**(1/2/3+개 선택) 대응. 특정 리포트가 아니라 **엔진에 배포**되어 **재생성 시 반영**.
- **§7 준수**: 융합 결과 문장에 원분야 단어(종교/교육/경영…)가 **직접 노출되지 않음**.
- **결정론적**: 같은 답변 → 같은 결과 (Math.random 금지, fingerprint 기반).

## 1. 왜 "고차원 수학"인가 — 벤치마킹 & 원리

### 1-1. 참고한 원리 (학습 대상)

| 원리 | 출처/개념 | 융합에 적용하는 방식 |
|---|---|---|
| **의미 벡터 공간 (semantic embedding space)** | word2vec / GloVe: 단어를 다차원 실수 벡터로 표현, 의미가 가까우면 벡터가 가까움 | 21분야를 **각각 D차원 속성 벡터**로 사전(dictionary) 정의 |
| **벡터 합성 (vector composition / additive compositionality)** | "king − man + woman ≈ queen" (Mikolov). 벡터 덧셈이 의미 합성을 근사 | N개 분야 벡터를 **가중 합** → 융합 중심 벡터(centroid) 산출 |
| **중심점 / 무게중심 (centroid)** | 다차원 점집합의 평균 = 대표점 | 선택 분야들의 벡터 평균이 "융합 정체성의 좌표" |
| **가장 가까운 이웃 (nearest-neighbor / argmax)** | 합성 벡터에 가장 가까운 어휘를 선택 | 융합 좌표에서 **핵심어(코어)·동사(동작)·산출(결실)** 를 각 축의 argmax로 결정 |
| **결정론적 해시 tie-break** | djb2/FNV — 이미 엔진에 존재 | 동점(tie)일 때 fingerprint로 갈라 **재현성 + 고유성** 동시 확보 |

> 핵심 통찰: **"융합 = 속성 벡터들의 무게중심 + 그 좌표를 사람 말로 복원(decode)"**.  
> 원분야 단어는 벡터로 들어갔다가 **좌표만 남고 사라진다** → §7 자동 준수.

### 1-2. 왜 나열이 아니라 융합이 되는가 (수학적 근거)

- 나열: `f(A,B,C) = label(A) + "·" + label(B) + "·" + label(C)` — 원소가 그대로 보존됨 (사용자가 거부).
- 융합: `f(A,B,C) = decode( centroid( vec(A), vec(B), vec(C) ) )` — 원소가 **좌표 연산에 흡수**되어 사라지고, 새 좌표에서 **새 단어**가 나옴.
- 예: 종교(신념/지킴) + 교육(가르침/성장) + 경영(조직/성과)  
  → centroid ≈ {가치=높음, 전달=높음, 구조화=높음}  
  → decode: 코어="신념", 동작="가르쳐", 결실="조직으로 키우는" → **"신념을 가르쳐 조직으로 키우는 일"**

## 2. 속성 공간 설계 (Attribute Space)

### 2-1. 축(axis) 정의 — 3축 × 각 축 어휘 슬롯

각 분야를 **3개 의미 축**의 좌표 + 각 축의 **대표 어휘 후보**로 표현한다.

| 축 | 의미 | decode 결과가 되는 문장 조각 |
|---|---|---|
| **CORE (코어/무엇을)** | 그 분야가 다루는 본질 대상 | 명사: "신념", "배움", "질서" … |
| **ACT (동작/어떻게)** | 그 분야의 핵심 행위 | 동사구: "가르쳐", "지켜", "이어" … |
| **FRUIT (결실/무엇으로)** | 그 분야가 남기는 산출 | 명사구: "조직으로", "다음 세대에게", "작품으로" … |

### 2-2. 21분야 속성 사전 (초안 — 확정 전, task 4에서 정밀화)

> 각 분야는 `{ core:[가중치 벡터], act:[…], fruit:[…] }` 형태.  
> 아래는 **개념 예시 3종**만 (전체 21종은 승인 후 사전으로 확정).

```
종교 → core:{신념/가치}, act:{지킨다/붙든다}, fruit:{삶의 방향으로}
교육 → core:{배움/성장},  act:{가르친다/전한다}, fruit:{다음 세대에게}
경영 → core:{질서/성과},  act:{세운다/이끈다},  fruit:{조직으로}
예술 → core:{아름다움},    act:{표현한다},        fruit:{작품으로}
… (총 21개)
```

### 2-3. 융합 산식 (결정론적)

```
입력: domains = [d1, d2, …, dN]  (선택 순서 = 자연 가중, domains[0] 최우선)
1) 가중치 w_i = 1 / (i+1)   ← 선택 순서 반영 (첫 분야 최대 가중) — 나열이 아니라 '무게'
2) 각 축(core/act/fruit)별로 후보 어휘에 Σ w_i·score 누적
3) 각 축의 argmax 어휘 선택 (동점 → fingerprint 해시로 tie-break, 재현성 유지)
4) 문장 골격에 대입:
   - N==1: "<core>을(를) <act> 일"                     예) "신념을 지키는 일"
   - N==2: "<core>을(를) <act> <fruit> 일"             예) "신념을 가르쳐 다음 세대에게 잇는 일"
   - N>=3: "<core>을(를) <act> <fruit> 키우는 일"       예) "신념을 가르쳐 조직으로 키우는 일"
5) 조사(을/를, 와/과)는 기존 _waGwa/_applyJosaMarkers 재사용
```

> **고유성**: 동일 분야 조합이라도 fingerprint tie-break + 문장 골격 변주로 사람마다 미세하게 달라짐.  
> **맞춤화**: 선택 순서(가중치)가 결과를 바꿈 → [종교,교육,경영] ≠ [경영,교육,종교].

## 6. 사용자 확인 요청 항목

1. **3축(CORE/ACT/FRUIT) 구조**로 가도 될지 — 혹은 축을 더/덜 두길 원하는지.
2. **선택 순서 = 가중치**(첫 분야 최우선) 규칙 동의 여부.
3. **N개별 문장 골격**(1개=핵심, 2개=중간, 3+개=확장) 문형 톤 확인.
4. 21분야 **속성 사전 어휘**는 승인 후 task 4에서 표로 확정 → 재확인받을지, 위임할지.

> 위 4개 확인되면 → task 4(사전 확정) → task 5(구현) → task 6(검증) → task 7(배포) 순서로 진행.

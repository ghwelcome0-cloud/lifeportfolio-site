| 종합 한눈에 보기 | **없음** (섹션을 다 읽어야 파악) | ❌ 인바디는 첫 장 요약 그래프 |

### 데이터 자산 (이미 보유 — 시각화 재료 충분)
- **4축 점수**: `self_understanding`·`self_expression`·`self_design`·`self_execution` 각 `content.pct` (0~100)
- **4축 tier**: `deep`(숙성) / `active`(활성) / `emerging`(발현) / `seed`(씨앗) + tierLabel
- **프로그램 효과**: `fitJob`(직무적합성) · `expansion`(직업확장성) · 분기테마 · 3주루틴 · 다음단계
- 결론: **데이터는 충분, 시각화 레이어만 없음.** → 신규 렌더만 추가하면 됨(엔진 무변경 가능).

## 1. 설계 철학 — "인바디를 넘는" 3원칙

> 인바디는 **"몸의 현재 상태 측정값"**을 보여준다.
> 인생포트폴리오는 **"고유한 나 + 나아갈 방향 + 살아낼 자산"**을 보여줘야 한다.
> → 단순 측정 그래프가 아니라 **"고유성을 한 눈에 + 방향을 함께 그리는"** 시각화.

| 원칙 | 인바디 | 인생포트폴리오(초월 지점) |
|---|---|---|
| **P-VIZ-1 한눈에(at-a-glance)** | 첫 장 종합 막대 | 첫 장에 **4축 레이더 + 종합 시그니처** — 나의 고유한 모양을 1초에 인지 |
| **P-VIZ-2 비교가 아닌 고유성** | 표준치 대비 비교 | **표준 비교 안 함** — "80억 중 단 하나의 모양". 균형/불균형이 곧 개성 |
| **P-VIZ-3 측정→방향** | 측정에서 끝 | 측정(4축) → **추천 진로·교육 → 비전(부채꼴)**까지 시각적으로 연결 |

### 왜 레이더(radar)인가 — 목적 최적화
- 4축은 **우열보다 "모양(균형)"이 곧 고유성**. 레이더는 모양을 1초에 보여줌 → P-VIZ-2 부합.
- 막대는 "어느 축이 높은가"(우열 강조)에 강함 → 보조로 함께 제공(축별 상세).
- **결론: 메인=레이더(고유한 모양) + 보조=막대(축별 tier·강도).** 둘 다 제공하되 레이더를 첫 인상으로.

## 2. 시각화 컴포넌트 설계 (TO-BE)

### [V1] 4축 레이더 — "나의 고유한 모양" (리포트 최상단 요약)
- 4축 pct를 꼭짓점으로 하는 4각 레이더.
- 각 꼭짓점 색 = tier 색(deep=teal, active=orange, emerging=indigo, seed=slate).
- 중앙에 **시그니처 한 줄**(이미 ①요약에 있음) 배치 → "모양 + 이름" 동시 인지.
- **표준치 비교선 없음**(P-VIZ-2). 대신 "이 모양은 당신만의 것" 캡션.

### [V2] 4축 강도 막대 — tier 신호 (각 self_ 섹션 카드 상단)
- 가로 막대 1개 + pct + tier 라벨 칩.
- tier 색상 신호: deep(짙은 teal) → active(orange) → emerging(indigo) → seed(slate).
- 기존 "응답 강도 72%" 텍스트를 **막대+칩**으로 격상(텍스트 보존, 시각 추가).

### [V3] 부채꼴 비전 맵 — "진로×교육→비전" (career_education 섹션)
- v1.5 directions 부채꼴을 **시각적 부채꼴**로: 진로/교육(아래) → 비전(위 꼭짓점), 갈래(subDirections)를 부채살로.
- 텍스트(이미 완성된 pathLine·subDirections)를 시각 구조로 감쌈.

### [V4] 프로그램 효과 게이지 (program.html)
- `fitJob`·`expansion`을 도넛/반원 게이지로. 3주루틴은 타임라인 바.

> **우선순위(이번 1차)**: V1(레이더) + V2(막대) = 리포트 핵심. V3·V4는 2차.

## 3. 기술 기준 (구현 제약)

| 기준 | 결정 |
|---|---|
| 라이브러리 | **순수 SVG 자작**(외부 의존 0) — PDF 캡처(html2canvas) 호환·로딩속도·오프라인 안전. Chart.js 미도입. |
| PDF 호환 | html2canvas가 SVG 렌더 가능하도록 inline SVG + 명시적 width/height. `svg script` 제거 로직(line 1053)과 충돌 없게 정적 SVG만. |
| 반응형 | viewBox + 100% width. 모바일 360px~데스크톱 대응. |
| 접근성 | `<title>`/`aria-label`, 색상+텍스트 병기(색맹 대응). |
| 비파괴(대원칙-B) | 기존 텍스트 표현 **삭제 금지**, 시각화는 **추가 레이어**. 데이터 없으면 시각화 skip, 텍스트 폴백 유지. |
| 결정성 | 같은 데이터 → 같은 그림(랜덤 0). |
| 엔진 무변경 | report-engine-v4.js 수정 없이 report.html 렌더 레이어만 추가(데이터 이미 충분). |

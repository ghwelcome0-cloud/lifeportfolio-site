## 🔴 RED — 필수 통과 항목 (1개라도 미통과 시 머지 금지)

### A. 한국 개인정보보호법 준수

- [ ] **§15 (수집·이용 동의)** Microsoft Clarity 활성화 전 **명시적·구체적 동의** 획득 절차 구현됨
- [ ] **§17 (제3자 제공)** Clarity = 위탁이므로 §17 직접 적용 안 됨, 그러나 §25 (위탁) 통지 의무 충족
- [ ] **§22 (동의 분리)** 약관 동의 / 처리방침 동의 / 마케팅 동의 / 분석 동의가 **각각 별도 체크박스**로 분리
- [ ] **§22 (사전 체크 금지)** 모든 선택 동의 체크박스의 기본값이 **미체크**
- [ ] **§22-2 (만 14세 미만 보호)** 가입 시 연령 확인, 만 14세 미만 차단 또는 법정대리인 동의 절차
- [ ] **§28-8 (국외 이전)** Microsoft Corporation (미국) 이전에 대한 다음 7가지 모두 명시:
  - [ ] 이전받는 자
  - [ ] 이전 국가
  - [ ] 이전 일시·방법
  - [ ] 이전 항목
  - [ ] 이전 목적
  - [ ] 보유 기간
  - [ ] 거부 권리·방법
- [ ] **§30 (처리방침 기재 사항)** 13개 필수 항목 모두 포함
- [ ] **§37 (처리정지·동의 철회)** 마이페이지에서 1-click 철회 가능

### B. 한국 전자상거래법 준수

- [ ] **§21 (금지행위)** 거짓·과장 광고 없음 — **정가 19,900원 거짓 표시 제거 완료** ✅
- [ ] **§17 (청약철회)** 디지털 콘텐츠 사전 고지 완료 (기존 §7 그대로 유지)
- [ ] **§13 (사업자 정보 표시)** 상호·대표·사업자번호·통신판매업번호·주소·연락처 표시 (기존 footer 유지)

### C. 정보통신망법 준수

- [ ] **§50 (영리목적 광고성 정보)** 마케팅 동의는 별도 옵트인 (기존 정책 유지)
- [ ] **§50-7 (사후동의 금지)** 가입 후 임의로 마케팅 동의 처리 안 함
- [ ] **§29 (취약 이용자 보호)** 만 14세 미만 차단 또는 법정대리인 동의

### D. 윤리·기독교 가치관 가드레일

- [ ] **다크 패턴 없음** — "동의"와 "거부" 버튼이 시각적으로 동등 위계
- [ ] **강제 묶음 동의 없음** — 선택 동의 거부해도 서비스 이용 가능
- [ ] **사전 체크 없음** — 모든 선택 동의는 기본 미체크
- [ ] **거짓 희소성 없음** — "3명 남음", "오늘만 할인" 류 없음
- [ ] **거짓 할인 없음** — 실제 미적용 정가 표시 없음 ✅ (완료)
- [ ] **수치심·자존감 공격 없음** — 카피 톤 점검 완료
- [ ] **부의 복음 변질 없음** — "사면 부자 됩니다" 류 없음
- [ ] **타 종교 비하 없음**
- [ ] **미성년자 표적 마케팅 없음**
- [ ] **취약계층 표적 마케팅 없음** (저소득·고령자·정신건강 취약자 등)

## 📋 PR 머지 전 최종 검증 절차

### Step 1: 자체 검증
- [ ] 본 체크리스트의 🔴 RED 항목 100% 통과
- [ ] 매핑표 v2의 모든 카피 변경 사항 적용

### Step 2: 외부 검토 (둘 중 하나 이상)
- [ ] **변호사 검토** — 개인정보보호 전문 변호사

### Step 3: 사용자 최종 승인
- [ ] 사용자가 변호사/KISA 검토 결과 확인 후 명시적 승인
- [ ] 승인 메시지/이메일 기록

### Step 4: 머지 + 사후 모니터링
- [ ] PR 머지 후 7일 내 사용자 문의·이의 모니터링
- [ ] 문제 발생 시 즉시 롤백 절차 준비

## 🚨 비상시 롤백 절차

만약 머지 후 법률·윤리 문제가 발견되면:

1. **즉시 Clarity 스크립트 비활성화** (`localStorage.clarity_consent = false` 강제 적용)
2. **이전 버전 처리방침/약관으로 git revert**
3. **이미 수집된 Clarity 데이터 일괄 삭제** (Microsoft 대시보드)
4. **사용자에게 공지** (이메일 또는 사이트 배너)
5. **사고 기록** — `/docs/incidents/YYYY-MM-DD-rollback.md` 작성

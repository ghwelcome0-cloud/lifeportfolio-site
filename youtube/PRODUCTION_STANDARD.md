# 인생 자산화 — YouTube 영상 제작 표준 (Production Standard)

> 1편·2편을 기준으로 확립한 제작 표준. **3편부터 모든 영상은 이 문서를 준수**한다.
> 목표: 매 편 동일한 "방송급 등급 A" 퀄리티 + 시리즈 톤 일관성.
> 최종 수정: 2편 완료 시점 (B-roll 개선안 v2 반영)

---

## 0. 제작 원칙 (불변)
- **비파괴**: 기존 파일 덮어쓰지 않음. 새 함수/새 파일로 작업 후 합침.
- **벤치마킹 먼저**: 새 기법 도입 전 방송급 다큐 레퍼런스 확인.
- **먼저 확인 후 적용**: 스크립트·디자인 변경은 사용자 확인 후 렌더.
- **크레딧 효율**: 재사용 가능한 자산(BGM·음성클론·design.py)은 재활용.
- **하나님 중심 세계관**: 모든 카피·썸네일은 겸손/정직/청지기 가치에 부합. 자기자랑·자극·세속적 허영 배제.

## 1. 제작 순서 (고정 워크플로)
```
① 블로그 분석  →  ② 벤치마킹  →  ③ 스크립트/8막 설계  →  ④ 사용자 확인
→  ⑤ 팩트체크  →  ⑥ 내레이션(클론음성)  →  ⑦ 슬라이드  →  ⑧ B-roll
→  ⑨ transcribe(자막타이밍)  →  ⑩ 타임라인(sections.json)  →  ⑪ 합성 빌드
→  ⑫ BGM 믹스  →  ⑬ SRT 생성  →  ⑭ 검수(신앙/팩트/비주얼)  →  ⑮ 메타+썸네일
```

## 2. 콘텐츠 구조 — 8막 (지식 다큐 표준)
| 막 | 역할 | 분량 |
|----|------|------|
| 1막 | 후크 (흐릿한 문제 제기 → 오늘의 열쇠 예고) | ~30s |
| 2막 | 핵심 개념 정의 + 3가지 핵심 | ~55s |
| 3막 | 과학적 근거 (연구 인용, 팩트체크 필수) | ~50s |
| 4막 | 변화/효과 (절제된 표현 + 윤리 안전장치) | ~50s |
| 5막 | 성경적 근거 (구약+신약 1구절씩) | ~58s |
| 6막 | 분별 (오해 경계 → 청지기 가치 재정립) | ~30s |
| 7막 | 온톨로지 (자기이해→표현→설계→실행 중 위치) | ~40s |
| 8막 | 핵심정리 + CTA (고정댓글 링크 유도) | ~40s |
- **총 길이 6~10분** (1편 5:42, 2편 6:02 기준). 너무 짧으면 B-roll 반복 위험 ↑.

## 4. 디자인 시스템 — design.py (1·2편 공통, 그대로 복사 사용)
**팔레트**
```
NAVY=(33,48,66)  NAVY_DK=(20,30,43)  NAVY_CARD=(44,62,84)
GOLD=(206,168,92)  GOLD_LT=(230,200,130)  GOLD_DK=(168,132,64)
WHITE=(244,247,251)  GREY=(150,165,182)  GREY_DK=(110,124,140)
```
**해상도/폰트**: 1920x1080. EB=font_eb(제목) / B=font_b(강조) / R=font_r(본문).
**필수 효과**: 비네트, gold glow(우상단), cool glow(좌하단), film grain(±4), 드롭섀도 텍스트, gold glow 텍스트, gradient 카드+섀도, gold 디바이더, **우하단 챕터 진행 점(chapter dots)**.
**핵심 함수**: `bg_base(chapter_idx, chapter_total)`, `text_shadow`, `text_glow`, `wrap_shadow`, `wrap_glow`, `tag`, `divider`, `card`, `save`.
- 슬라이드 개수 = TOTAL(챕터 점 개수)에 맞춤. 2편은 15개.

## 5. B-roll (AI 생성) — ★ v2 개선안 (3편부터 필수 적용)
**생성**: fal-ai/bytedance/seedance-2.0, 16:9, 5s, fast_mode=true. **전부 faceless/추상**. 얼굴·인물·텍스트 금지.
- 콘텐츠 모더레이션 주의: "river of light/streaming" 류 표현 거부될 수 있음 → 부드럽게 reword.
- 막별 의미 연결(예: 안개=흐릿함, 잉크=꺼냄, 펜=쓰기, 책=성경, 촛불=분별, 흐름=과정).

**🔴 v2 개선 — 반복 흐름 제거 (2편의 유일한 약점 보완):**
2편 문제: 5초 B-roll을 최대 4.3회 단순 loop → b3_pen·b6_flow 구간에서 같은 동작 반복이 눈에 띔.
해결(방송급 다큐 표준 기법):
1. **Ping-pong loop** — 정방향+역방향 concat → 이음새 없이 10초 단위. `[0:v]reverse[r];[0:v][r]concat=n=2:v=1:a=0[pp]`
2. **감속 0.7~1.0x** — `[pp]setpts=1.3*PTS` 로 더 길게 + 차분한 다큐 톤.
3. **느린 Ken Burns** — `zoompan` 미세 줌(1.0→1.06)으로 매 순환 화면이 미묘하게 달라짐. (선택: 사용자가 모션 싫어하면 생략)
4. **분할 배치** — 한 구간이 12초 넘으면 B-roll 하나로 채우지 말고 슬라이드와 번갈아.
- 규칙: **B-roll 단일 구간이 원본 길이(5s)의 2배(10s)를 넘으면 ping-pong+감속을 반드시 적용.**

**톤 매칭**: 빌드 시 1920x1080/30fps 업스케일 + navy 톤 오버레이
`eq=brightness=-0.06:saturation=0.92:contrast=1.04, colorbalance=rs=-0.04:bs=0.06:gs=-0.01`

## 6. 타임라인 (sections.json)
- 형식: `[[start, end, "key"], ...]` — key는 슬라이드명 또는 `"BROLL:파일명"`.
- transcribe(whisper-1)로 막별 정확한 타임스탬프 확보 후 매핑.
- 갭 없이 연속: 각 클립 = 다음 시작 - 현재 시작. 마지막은 narration 총길이.
- 슬라이드는 충분히 보이게, B-roll은 의미 맥락에 인터컷.

## 7. 합성 빌드 (build_video.py)
- 슬라이드: 정지 + fade in/out 0.5s (줌/팬 없음 — 사용자 요청).
- B-roll: §5 v2 기법 적용 + fade in/out.
- 클립 concat → video_silent.mp4 → 오디오 합성.
- FPS=30, libx264, yuv420p, +faststart.

## 8. 오디오/사운드
- 내레이션: loudnorm I=-16:TP=-1.5:LRA=11 (YouTube 표준 -16 LUFS).
- BGM: **1편 bgm.mp3 재사용** (시리즈 일관성+크레딧 절약). volume≈0.10, loop, 앞2s/뒤2.3s 페이드.
- 최종 mix 후 다시 loudnorm -16. ebur128로 -16.0 LUFS 검증 필수.

## 9. 자막 (SRT, ★ 별도 파일)
- whisper 세그먼트 → SRT (영상에 굽지 않음, 별도 .srt 제공).
- **인식 오류 교정 필수**: 2편에서 "시편"→"10편" 오인식 → 수동 교정. 성경 인용·고유명사 전수 확인.
- **클론 발음 검수 필수**: 클론 음성이 조사·축약을 기계적으로 발음할 수 있음(2편: "나에 대해"→"내 대해"). transcribe로 발음 확인 후, 문제 시 해당 막 텍스트를 자연스럽게 풀어써(예: "나 자신에 대해") 재생성. ★ 막은 1막=p1(1+2막 묶음) 식으로 묶여 있으니, 한 파트 재생성 시 그 파트의 전체 범위를 함께 재생성할 것(일부만 만들면 내용 누락).

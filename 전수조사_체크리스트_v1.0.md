## ⚡ Part A. 자동 점검 (스크립트 1줄로 완료 — 크레딧 거의 소모 없음)

```bash
cd /home/user/webapp && bash scripts/audit_check.sh
```

이 스크립트가 **자동으로 검사하는 23개 항목**:

| # | 점검 항목 | 회귀 시 증상 |
|---|-----------|-------------|
| 1 | 전체 46개 HTML JS 문법 오류 | 페이지 전체 동작 불능 |
| 2 | firebase.json + i18n JSON 유효성 | 배포 실패 / 다국어 깨짐 |
| 3 | Firebase 프로젝트 ID(`lifeporfolio`, t 누락이 정상) 일관성 | 인증/DB 연결 실패 |
| 4 | RTDB URL(asia-southeast1) 일관성 | 데이터 조회 실패 |
| 5 | **Permissions-Policy 헤더 문법** (origin 개별 따옴표) | 콘솔 경고 다발 |
| 6 | **결제 CSP 도메인** (payple 이원화 + payment-v2) + product-v2 meta에 form-action 없음 | **카드결제 차단** |
| 7 | **마이페이지 REST 우선(PR#181) + allSettled 미사용** | 로딩 수십초 / 리포트 미표시 |
| 8 | **검사 진입 쳇바퀴 방지(PR#192 _active)** | 모바일 '이어서 하기' 무한루프 |
| 9 | 결제→검사 플로우 9개 핵심 파일 존재 | 플로우 단절 |

> ✅ **"통과 N건 / 실패 0건 🎉"** 나오면 구조적 회귀 없음. 실패 시 해당 항목 반드시 수정 후 배포.

## 👁️ Part B. 수동 점검 (사람이 직접 — 자동화 불가 영역)

### B-1. 결제 플로우 (가장 중요 — 매출 직결) 🔴
- [ ] 홈 → "결제하기" → **product-v2**(KRW/페이플)로 이동하는가?
- [ ] `/product` 직접 접속 시 KO 사용자는 product-v2로 자동 이동하는가? (EN은 PayPal 유지)
- [ ] **PC 카드 결제** 결제창이 정상 노출·완료되는가?
- [ ] **PC 계좌이체** 정상 완료되는가?
- [ ] **모바일 카드 결제** 결제창이 about:blank 없이 정상 뜨는가?
- [ ] 결제 완료 후 payment-success → 검사 진입까지 자연스럽게 이어지는가?
- [ ] (EN) `?lang=en`에서 product.html PayPal(USD) 결제 정상인가?

### B-2. 검사(suvey) 플로우 🔴
- [ ] 결제 직후 검사 페이지로 **버퍼링 없이 즉시** 진입하는가? (fast-path)
- [ ] **모바일 '이어서 하기'** → 마이페이지로 안 튕기고 검사로 진입하는가? (쳇바퀴 없음)
- [ ] 검사 중간 저장 후 재진입 시 답변이 유지되는가?
- [ ] 검사 제출 → report-loading → report 정상 생성되는가?

### B-3. 마이페이지 🟡
- [ ] 로그인 후 로딩이 **1~2초 내** 완료되는가? (수십초 X)
- [ ] 완료된 리포트 목록이 정상 표시되는가?
- [ ] 진행 중 검사 목록이 정상 표시되는가?
- [ ] 진행 중 검사 '삭제' 버튼 동작 정상인가?
- [ ] (계정 확인) 리포트가 안 보이면 **올바른 계정으로 로그인했는지** 먼저 확인 (계정별 데이터 독립)

### B-4. 콘솔 청결도 (F12 → Console) 🟡
- [ ] 핵심 페이지(홈/결제/검사/마이페이지/리포트)에 **빨간 Error(Uncaught) 0건**인가?
- [ ] Permissions-Policy 경고 없는가?
- [ ] CSP 위반(blocked) 로그 없는가? (report-only 정보성 로그는 무방)
- ℹ️ `cdn.tailwindcss.com should not be used in production`은 **정상 안내**(무시 가능).

### B-5. 다국어(i18n) 🟢

- [ ] `?lang=ko` 복귀 시 한글 정상인가?
- [ ] 리포트 언어가 검사 제출 언어와 일치하는가?

### B-6. 반응형/모바일 UX 🟢

- [ ] 카카오톡 인앱브라우저에서 진입 정상인가?

## 🛡️ Part C. 절대 하지 말 것 (회귀 유발 금지 사항)

이 세션에서 실제 회귀를 일으킨 패턴들 — **재발 방지**:

1. ❌ **product-v2.html meta CSP에 `form-action` 추가 금지** → 카드결제 차단 재발.
   (meta는 폼 무제약, HTTP 헤더 CSP의 form-action만 적용하는 게 정답)
2. ❌ **마이페이지 리포트 로딩을 `Promise.allSettled`로 병렬화 금지** → 리포트 미표시 회귀.
   (직렬 await 유지)
3. ❌ **RTDB 조회를 SDK get() 우선으로 되돌리기 금지** → asia-southeast1에서 20초 timeout.
   (REST 우선 유지 = PR#181)
4. ❌ **suvey 진입 게이트에서 `_active`(진행중 세션) 확인 제거 금지** → 모바일 쳇바퀴 재발.
   (PR#192 유지)
5. ❌ **`git add -A` / `git add .` 사용 금지** → 항상 `git add <파일명>` 명시적으로.
6. ❌ **프로젝트 ID `lifeportfolio`로 수정 금지** → 정답은 `lifeporfolio`(t 누락이 의도된 정상).

## 📋 Part D. 배포 절차 (표준)

1. 코드 수정
2. `bash scripts/audit_check.sh` → **통과 확인** (실패 시 수정)
3. `git add <수정파일>` (명시적, `-A` 금지) → `git commit -m "..."`
4. `git push origin main` (Hosting 자동배포 ~2.5~5분)
5. `curl -sL "https://lifeportfolio.co.kr/..."` 로 배포 반영 확인
6. Part B 수동 점검 (특히 결제/검사 플로우)
7. ⚠️ **Functions 수정 시**: 사용자 로컬 PowerShell에서 별도 배포 필요

## 🔄 사용 방법 (사용자 → AI 제공용)

주기 점검 시 이 문서를 그대로 AI에게 제공하며:
> "전수조사 체크리스트로 점검해줘"

라고 요청하면, AI가:
1. `scripts/audit_check.sh` 실행 (Part A 자동)
2. 핵심 페이지 콘솔 점검 (Part B-4)
3. 발견된 문제를 Part C 위반 여부와 대조
4. 회귀 없이 수정 → 배포 → 검증
